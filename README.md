# カスタム休憩管理システム

長時間の連続稼働を防ぎ、適切な休憩サイクルを促すための macOS 向けデスクトップアプリです。  
既存のログ機能とは独立したプロセスとして動作することを想定しています。

## 目的

- 連続作業による疲労の蓄積を防ぐ
- 休憩を習慣化し、集中力と作業品質を維持する
- 強制休憩モードで「休む時間」を確保する

## 時間設定（本リポジトリの標準）

- 作業時間: **90分**
- 休憩時間: **15分**

> 補足: 要件定義書には別の初期値案（52分/17分）がありますが、本リポジトリでは運用方針として **90分/15分** を採用します。

## 想定機能

1. 作業時間のカウントダウン（バックグラウンド実行）
2. 作業終了時の休憩提案ダイアログ表示  
   - 表示文言: "Would you like to start your break now?"  
   - ボタン: "Yes" / "No (snooze)"
3. "Yes" 押下で休憩モード（全画面オーバーレイ）へ移行
4. 休憩モード中の残り時間表示（画面中央）
5. 休憩終了時にロック解除し、次の作業サイクルを自動開始
6. 緊急解除機能（フェイルセーフ）
7. セッション開始/停止の手動切り替え（仕事中のみ有効化）

## UI/UX方針

- 休憩提案ダイアログは最前面表示しつつ、作業を過度に妨げない設計
- 休憩モード画面は薄グレーの半透明オーバーレイを採用し、背面アプリ（例: Chrome）がうっすら見える設計
- UI文言は英語に統一し、シンプルで視認性の高いレイアウトを採用

## 推奨技術スタック

- 言語: Python
- GUIライブラリ: Tkinter または PyQt
- 対象OS: macOS

## 実行方法

1. Python 3 をインストール
2. プロジェクト直下で以下を実行

```bash
python3 app.py
```

### 休憩に入る瞬間をすぐ確認する（デモモード）

以下を実行すると、短時間で挙動を確認できます。

```bash
python3 app.py --demo
```

- 作業時間: 20秒
- 休憩時間: 15秒
- 延長時間: 10秒
- 緊急解除長押し: 2秒

## 現在の実装仕様（MVP）

- 起動直後は `IDLE`（待機状態）
- `Start Work Session` で 90 分作業カウントダウンを開始
- 時間到達時に英語ダイアログを表示（Yes: break / No: snooze）
- `Yes` で 15 分休憩モードへ移行（全画面・半透明オーバーレイ）
- `No` またはダイアログを閉じると 5 分延長
- 休憩終了で自動的に作業モードへ戻る
- 仕事終了時は `Stop Session` で停止可能
- 緊急解除は休憩画面で `Esc` キーを長押し

## 今後の実装メモ

- サイクル設定値（90/15）は設定ファイル化し、将来的に可変にする
- 緊急解除の誤作動を防ぐため、長押し時間や多段確認を検討する
- 通知頻度や延長回数のログを取り、運用改善につなげる
